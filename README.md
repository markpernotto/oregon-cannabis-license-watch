# Oregon Cannabis License Watch

## What this is, in plain English

Oregon publishes a list of every business with a license to grow, process,
sell, or test cannabis. The list is public, but it only ever shows what's
true *right now* — once a business disappears, the public record of it being
there is gone. There's no built-in "what changed this week?"

This project fixes that, modestly. Every night a small program downloads
the current list, compares it to yesterday's, and writes down what's
different — new licenses, removed licenses, name changes, expiration
renewals. After a while you have a real history.

The actual subject (cannabis licensing) matters less than the *pattern*:
"reliably watch a public dataset over time and notice when it changes."
That same pattern shows up in government contracts, FDA approvals,
building permits, lobbying disclosures, and a hundred other places. This
is a small, careful example of it.

I'm learning data engineering by building this. It is not perfect, the
schema will probably change, and there are corners I'd do differently if I
started over. The point is to *do* the work, not to ship the world's most
polished pipeline. If something here is useful to you, great; if you spot
something I got wrong, please open an issue.

## Status

Phase 1 backend is complete and running nightly against live OLCC data.
The React UI works locally and reads the same JSON the public feed uses.
The five-day green-nightly streak is in progress.

## How it works (the short version)

```
OLCC website  ──► extract ──► transform ──► load ──► diff ──► publish
(public data)     (download)   (clean up)   (DB)     (compare)  (write JSON+RSS)
                                                                       │
                                                                       ▼
                                                             web UI · RSS readers
```

Five small pieces, each one easy to explain. Detailed pipeline diagram
in [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Public outputs

- **`public/changes.json`** — last 30 days of changes, structured.
- **`public/rss.xml`** — same data, RSS-feed flavor, subscribe in any reader.
- **`data/snapshots/YYYY-MM-DD.csv`** — the raw nightly snapshot, archived.

These files are regenerated by a GitHub Action on a nightly cron and
committed back to the repo. **Freshness SLO: published data is no more
than 26 hours stale from source.**

## Run it locally

```bash
# 1. Python pipeline
cp .env.example .env
python3.12 -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'

# Local Postgres (any Postgres 16+ works; docker compose up db included)
psql "$DATABASE_URL" -f etl/schema.sql

# Run the full pipeline against live OLCC
python -m etl.run

# Run the test suite
pytest                                      # unit tests only
DATABASE_URL=... pytest                     # adds integration tests

# 2. Web UI
cd web
npm install
npm run dev   # http://localhost:5199/
```

## What's where

- `etl/` — the five pipeline modules (extract, transform, load, diff, publish)
  plus `run.py` (CLI), `vocab.py` (controlled-vocabulary loader), and
  `schema.sql`. The TLS chain workaround lives in `etl/certs/` because OLCC's
  server doesn't ship the intermediate certificate; see the README there
  for the why.
- `vocabularies/` — controlled vocabularies for license type, status, and
  change type. SKOS-lite YAML. Kept versioned because vocabularies drift.
- `web/` — Vite + React + TypeScript. Reads `public/changes.json` directly.
- `data/snapshots/` — daily CSVs (archival series).
- `docs/` — architecture, data catalog, data sources, Tableau-extraction
  research notes.
- `tests/` — 27 tests. Unit tests run in CI; integration tests need a
  Postgres at `$DATABASE_URL`.

## Data sources

This phase tracks one source: the OLCC Cannabis Licensee public report.
Future phases will add tax distribution PDFs, market data, theft reports,
and more. Inventory in [docs/DATA_SOURCES.md](docs/DATA_SOURCES.md);
catalog entry per dataset in [docs/DATA_CATALOG.md](docs/DATA_CATALOG.md).

## Principles

This is a republishing project. It does not enrich, score, rank, or join
public data against other sources to reconstruct information OLCC chose
not to publish. Full statement in
[CONTRIBUTING.md](CONTRIBUTING.md#principles); legal/privacy posture in
[PRIVACY.md](PRIVACY.md).
